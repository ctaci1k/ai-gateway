# PH22 — Суддя оцінює доданий AI · тонкий скрол сайдбара · base_url для дефолтних слотів

> Три проблеми, виявлені під час тесту BYOK. Дві — бекенд (суддя), дві — фронт.
> Enterprise-рівень, атомарні кроки, гейти після кожного. Коміт/пуш — лише на
> пряме прохання власника.

## Проблеми (зі слів власника, 2026-05-31)
1. **Суддя не оцінює 4-й доданий AI** — ставить йому 0/100 і ніколи не обирає.
2. **Скрол лівої частини (сайдбар)** — коли вміст не вміщається, з'являється
   звичайний (дефолтний) скролбар, а не наш тонкий тематичний.
3. **Не можна замінити дефолтний слот (AI 3) своїм ключем іншого провайдера** —
   бо в дефолтного слота base_url зашитий (sambanova), а ключ від іншого
   провайдера (Gemini/OpenRouter) → не працює. Треба дати можливість задати
   base_url і для дефолтних слотів.

## Контекст із коду (ground truth, звірено 2026-05-31)

### Проблема 1 — суддя й доданий AI (КОРІНЬ знайдено)
- **Промпт судді хардкодить 3 моделі.** `backend/prompts/prompts.yaml`
  (`provider_selector_system`, ~ряд 116–134): приклад `scores` лише
  `{groq,cerebras,sambanova}` і пряма інструкція: «**The selected_model field MUST
  contain ONLY: groq, cerebras, or sambanova.**» → суддя ігнорує/занулює доданий
  AI і не може його обрати.
- **Парсер відкидає не-дефолтний вибір.** `backend/selector/selector_parser.py:14`:
  `if selected_model not in ALLOWED_MODELS: selected_model = ""`, де
  `ALLOWED_MODELS = [groq, cerebras, sambanova]` (`config/selector_config.py`). Тобто
  навіть якщо суддя обере доданий слот — парсер його занулює.
  (`response_selector.select_best_response` уже рахує `allowed = set(responses.keys())`
  — реальний набір, тож фільтр у парсері і зайвий, і шкідливий.)
- **Криптоназва в промпті.** `selector_prompt.py::build_selector_prompt` підписує
  кожну відповідь `MODEL NAME: <provider_name>`; для доданого слота це
  `custom-xxxxxx` (слаг) — суддя бачить безглузду назву. Відповідь доданого AI
  **доходить** до судді (`responses` = всі `all_responses`, включно з custom).
- **Симптом 0/100 на фронті** — наслідок: суддя не повертає бал для custom-слота →
  `selector_scores` без нього → картка показує 0 (`components/compare/ModelScoreCard.tsx`
  `/100`, `CompareTurn.tsx` мапить `selector_scores`).

### Проблема 2 — скрол сайдбара
- `theme/components.css`: `.sidebar` (ряд 15) має `overflow-y: auto` (ряд 24), але
  **не** входить у групи тонкого скролу `.thin-scroll`/`.msgs`/… (ряд ~489–520).
  Тож використовується дефолтний скролбар. (Аналогічно вже зробили для
  `.keys-dialog` у PH18.)

### Проблема 3 — base_url для дефолтних слотів
- **Бекенд уже підтримує.** `provider_service._transient_base_url(slot, base_url)`:
  `base_url or DEFAULT_BASE_URLS.get(slot)` — якщо дефолтному слоту передати
  base_url, він використається; `/keys/validate` теж (через `build_transient_responder`).
- **Фронт обмежує.** `KeysModal` показує поле Base URL лише для `r.custom`;
  `KeysContext.saveAndValidate` шле `base_url: r.custom ? r.baseUrl.trim() || undefined
  : undefined`; `byokPayload` так само лише для custom. Тобто дефолтний слот не може
  отримати base_url з UI.
