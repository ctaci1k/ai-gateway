## Легенда статусів

У всіх документах статус реалізації позначається так:

- ✅ **Реалізовано** — є в коді, працює.
- 🟡 **Частково** — є частина, потребує доробки.
- ⛔ **Не реалізовано** — описано, але в коді відсутнє.
- 🔮 **Майбутнє** — свідомо відкладено на потім.

## Найважливіше (TL;DR)

1. Це **AI Gateway / Router**, а не звичайний чатбот: один запит → кілька моделей → AI-суддя обирає найкращу відповідь → персоналізація під користувача.
2. **Реальні відповідачі** — `groq`, `cerebras`, `sambanova`. **Суддя (selector)** — `gemini`. Це відрізняється від старого опису (GPT/Claude/Gemini) — див. [10-open-decisions.md](10-open-decisions.md).
3. Багато з первісного бачення (RAG, БД, мульти-юзер, Docker/VPS) **ще не побудовано**. Ground truth — у [08-current-state.md](08-current-state.md).
4. Оригінальний brain-dump збережено в [archive/ORIGINAL-braindump.txt](archive/ORIGINAL-braindump.txt) як історичне джерело.

## Принципи цих документів

- **Не вигадувати.** Усе про «поточний стан» — прив'язане до коду.
- **Розділяти шари.** Бачення ≠ реальний стан ≠ план. Не змішувати.
- **Контракти, а не лозунги.** Замість «система обробляє помилки» — конкретні схеми та коди відповідей.
- **Пріоритети явні.** MVP відокремлено від «nice-to-have».
