PEŁNY FUNKCJONAL PROJEKTU
Personalized Multi-Model AI Gateway / AI Router

1. GŁÓWNA IDEA SYSTEMU
System jest web application z AI chatem, która:
działa online przez własny serwer,
obsługuje wiele modeli AI jednocześnie,
pobiera wiele odpowiedzi na jeden prompt,
wykorzystuje oddzielny AI Selector do wyboru najlepszej odpowiedzi,
pozwala użytkownikowi porównywać odpowiedzi modeli,
zapamiętuje preferencje użytkownika,
działa jako personalized AI system.
Projekt NIE jest zwykłym chatbotem.
To:
AI Gateway,
AI Router,
Multi-Model Assistant,
Personalized AI System.

2. WEB INTERFACE
Użytkownik korzysta z systemu przez przeglądarkę.
Frontend zawiera:
chat UI,
sidebar z chatami,
input promptów,
przycisk Send,
przycisk New Chat,
przycisk Delete Chat,
przycisk Show All Responses,
upload plików/PDF,
streaming odpowiedzi.
Frontend budowany jest przy użyciu:
React,
Next.js,
Tailwind CSS.

3. CHAT SYSTEM
System obsługuje:
tworzenie nowych chatów,
przełączanie między chatami,
usuwanie chatów,
osobną historię dla każdego chatu,
persistent chat history,
session isolation.
Każdy użytkownik posiada:
własne chaty,
własną pamięć,
własne preferences,
własny context history.

4. MESSAGE SYSTEM
Użytkownik może:
wysyłać wiadomości,
usuwać wiadomości,
czyścić historię,
czyścić responses,
czyścić all_responses.

5. MULTI-MODEL REQUEST SYSTEM
Backend równolegle wysyła prompt do:
GPT,
Claude,
Gemini,
przyszłych modeli AI.
Zapytania wykonywane są async przez:
asyncio,
httpx,
LiteLLM.

6. RESPONSE STORAGE
System przechowuje:
user_message,
best_response,
selected_model,
all_responses.
Przykład:
responses = {
   "gpt": "...",
   "claude": "...",
   "gemini": "..."
}

7. ALL RESPONSES VIEW
Użytkownik może:
widzieć tylko best_response,
lub otworzyć wszystkie odpowiedzi modeli.
Frontend posiada:
Show All Responses

8. AI SELECTOR SYSTEM
System posiada oddzielny AI Selector.
Selector AI:
otrzymuje wszystkie odpowiedzi,
analizuje je,
wybiera best_response.
Selector ocenia:
poprawność,
kompletność,
styl,
czytelność,
jakość wyjaśnienia,
zgodność z preferencjami użytkownika.

9. USER PREFERENCE MEMORY
System zapamiętuje:
które modele użytkownik wybiera,
które odpowiedzi otwiera,
jaki styl odpowiedzi preferuje.
Preferences wykorzystywane są przez:
AI Selector,
response ranking system.

10. PERSONALIZED RESPONSE SELECTION
AI Selector uwzględnia:
previous preferences,
preferred response style,
history model selections.
System personalizuje wybór odpowiedzi pod użytkownika.

11. CHAT MEMORY SYSTEM
System posiada:
ChatBuffer
ChatBuffer:
przechowuje historię,
działa przez sliding window,
automatycznie usuwa stare wiadomości,
serializuje dane do JSON.

12. SLIDING WINDOW SYSTEM
Jeśli historia rozmowy przekroczy limit:
najstarsze wiadomości są automatycznie usuwane.
System:
ogranicza memory usage,
ogranicza token usage,
stabilizuje backend.

13. PROVIDER ARCHITECTURE
Każdy model działa przez provider layer:
OpenAIProvider,
ClaudeProvider,
GeminiProvider,
future providers.

14. BASE PROVIDER CONTRACT
Wszystkie provider’y:
posiadają wspólny interfejs,
implementują generate(),
obsługują streaming,
działają przez BaseProvider.

15. LITELLM INTEGRATION
LiteLLM:
standaryzuje provider layer,
umożliwia wspólną obsługę modeli,
upraszcza multi-model orchestration.

16. STREAMING RESPONSES
Odpowiedzi mogą być streamowane:
token-by-token,
jak w ChatGPT.
Frontend aktualizuje odpowiedź w czasie rzeczywistym.

17. RAG SYSTEM
System obsługuje:
upload PDF,
upload dokumentów,
upload plików tekstowych.
Backend wykonuje:
chunking,
embeddings,
vector search,
retrieval context injection.

18. VECTOR DATABASE
System wykorzystuje:
ChromaDB.
Vector DB przechowuje:
embeddings,
chunks,
metadata,
retrieval indexes.

19. DATABASE SYSTEM
Projekt wykorzystuje:
SQLite (start),
PostgreSQL (future upgrade).
Baza danych przechowuje:
users,
chats,
messages,
preferences,
sessions.

20. ORM
Projekt wykorzystuje:
SQLAlchemy
Do:
modeli bazy danych,
relacji,
persistent storage.

21. FASTAPI BACKEND
Backend działa przez:
FastAPI
Główne endpointy:
/chat
/new-chat
/delete-chat
/all-responses
/upload-document

22. JSON API
Frontend i backend komunikują się przez:
JSON requests,
JSON responses.

23. SECURITY SYSTEM
System posiada:
request limits,
spam protection,
safe validation,
safe file upload,
memory cleanup,
history limits,
error isolation.

24. ERROR HANDLING
Backend:
nie może crashować przy błędach modeli,
obsługuje failed responses,
zwraca stabilne odpowiedzi użytkownikowi.

25. PERSISTENT STORAGE
Dane:
nie znikają po restarcie,
są przechowywane permanentnie.

26. DOCKER DEPLOYMENT
System działa przez:
Docker,
Docker Compose.

27. UBUNTU VPS DEPLOYMENT
Projekt jest przygotowany pod:
Ubuntu VPS,
production deployment,
public access.

28. NGINX + SSL
Deployment zawiera:
Nginx reverse proxy,
HTTPS,
SSL,
domain support.

29. TESTING
Projekt posiada:
unit tests,
ChatBuffer tests,
selector tests,
aggregation tests,
API tests.

30. FINALNY CEL SYSTEMU
Projekt jest:
production-like AI application,
multi-model AI gateway,
personalized AI assistant,
AI orchestration platform,
scalable web AI system.
